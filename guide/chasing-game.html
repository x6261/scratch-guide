The “forever” block in the Control tab repeats any code inside it until the project stops. Forever loops are necessary in most games because they keep essential actions running at all times—such as moving characters, updating health or score, checking for collisions, playing animations, and responding to player input. Without a forever loop, these continuous game behaviors wouldn’t function properly.

Also in the Control tab, you’ll find the “if () then” block, which is called a conditional. It runs the code inside only when the condition in the hexagon-shaped slot is true. Conditionals are essential for decision-making in games, such as checking whether a key is pressed, an enemy is touched, or a score reaches a certain value.
